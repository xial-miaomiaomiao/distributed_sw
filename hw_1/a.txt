## 1. 容器环境
- 使用 Docker 容器化部署整个系统
- 包含：数据库、后端服务、Nginx 三个容器
- 需要编写 Dockerfile 和 docker-compose.yml

## 2. 负载均衡
- 后端服务启动多个实例（如 8081、8082 端口）
- 通过 Nginx（80端口）进行反向代理和请求转发
- 配置不同的负载均衡算法（轮询、权重、IP哈希等）
- 使用 JMeter 压测，验证请求是否均匀分配到各个后端

## 3. 动静分离
- 编写简单的前端页面（HTML + CSS + JS）
- Nginx 配置动静分离：静态文件直接由 Nginx 返回，动态请求转发给后端
- 分别压测静态文件和后端服务，对比响应时间

## 4. 分布式缓存
- 引入 Redis 作为缓存层
- 实现商品详情页的缓存功能
- 处理缓存的三大问题：
  - **缓存穿透**：查询不存在的数据，直接打到数据库
  - **缓存击穿**：热点缓存过期，大量请求同时打到数据库
  - **缓存雪崩**：大量缓存同时过期，数据库压力剧增
---
**整体目标**：构建一个能应对高并发读请求的分布式系统，通过负载均衡、动静分离、缓存等技术提升系统性能和可用性。
